# Entrega 12 - Planejamento de Avaliação

### A) Planejamento de Usabilidade (Método DECIDE)

| Letra | Descrição |
| :---: | :--- |
| D | O objetivo é avaliar se o nosso público-alvo, como estudantes e jornalistas, consegue enviar imagens sem dificuldade e entender claramente se o resultado aponta um deepfake ou não. Queremos descobrir onde o usuário trava ou se confunde. |
| E | Vamos investigar se o botão de upload está fácil de achar e se a pessoa entende de imediato a porcentagem de confiança da análise. Também queremos saber se as mensagens de erro são claras, como quando o arquivo é muito grande, e se a navegação até a área de ensino é simples. |
| C | Vamos usar Testes de Usabilidade observando pessoas reais usando o sistema e aplicar questionários no final para saber a opinião delas. Antes disso, a própria equipe fará uma inspeção técnica na interface para corrigir erros de design mais óbvios. |
| I | Os participantes serão pessoas que representam nossas personas, como estudantes de comunicação e usuários de redes sociais. Para os testes online, vamos precisar apenas de computadores ou celulares com internet e uma ferramenta para gravar a tela da sessão. |
| D | A avaliação será feita preferencialmente online, via Google Meet, ou presencialmente. O participante terá que cumprir tarefas como enviar uma foto suspeita, ler o relatório e acessar o material educativo. Tudo deve durar entre 15 e 20 minutos, sempre com autorização prévia e garantia de anonimato. |
| E | Na análise, vamos verificar se o usuário conseguiu completar as tarefas, quanto tempo levou e quais foram as reclamações ou dúvidas que surgiram. Com esses dados, vamos criar uma lista de prioridades para arrumar no protótipo. |

### B) Lista de Instrumentos:

i) Termo de consentimento (TCLE) – Documento que explica para que serve o teste, garante que a identidade da pessoa será preservada, avisa que ela pode desistir quando quiser e pede permissão para gravar a sessão.

ii) Questionário – Será aplicado em dois momentos: antes do teste, para entender o nível de conhecimento do usuário sobre deepfakes, e depois, para medir a satisfação e facilidade de uso.

iii) Roteiro e Tabela de observação – Um guia com o passo a passo que o usuário deve seguir e um espaço para anotarmos as dificuldades, o tempo gasto e os erros que acontecerem durante o uso.

iv) Formulário de avaliação heurística – Uma lista baseada nas regras de Nielsen que a equipe técnica vai usar para inspecionar a interface e corrigir problemas antes de chamar usuários reais.
